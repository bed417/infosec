a) Read and summarize (with 1-5 bullet points for each heading)

Schneier 2015: Applied Cryptography Chapter 1: Foundations 

        1.1 TERMINOLOGY

o   Sender and receiver

  sender wants to get the message to the receiver securily, making it impossible for eavesdroppers to read

o   Messages and encryption

  Plaintext [Encryption] Ciphertext [Decryption] Original Plaintext

  Cryptograhpy; keeping messages secure

  Cryptanalysis; breaking ciphertext

  Cryptology; mathematics studying cryptography and cryptanalysis

o   Authentication, integrity, and nonrepudiation

  Authentication; ascertain messages origin

  Integrity; verify message has not been modified

  Nonrepudiation; sending message cannot be falsely denied

o   Algorithms and keys

  Algorithm/cipher; mathematical function used for encryption and decryption

  Key; used by nonrestricted algorigthms for encryption and decryption

  Good algorithm is open for scrutiny; key makes it secure

o   Symmetric algorithms

  encryption key can be calculated from the decryption key and vice versa

  often using the same key for encryption and decryption

  security dependent on keeping the key secret

  key logistics difficult

o   Public-key algorithms

  Using different keys for encryption and decryption

  Message is encrypted with public key; decrypted with private key

  Message is signed with private key; signature verified with public key

o   Cryptanalysis

  science of recovering plaintext without access to the key

  attack = attempted cryptanalysis

o   Security of algorithms

  degrees of security vary

  important that the value of the data remain less than the cost to break the security

  Good cryptosystems take into consideration the evolution of computing power

o   Historical terms

  code; cryptosystem dealing with linguistic units such as words or phrases

        1.2 STEGANOGRAPHY

o   hides secret messages in other messages, such as images

        1.3 SUBSTITUTION CIPHERS AND TRANSPOSITION CIPHERS

o   Substitution ciphers; each character in the plaintext is substituted for another character in the ciphertext

o   Transposition ciphers; plaintext remains the same, but the order of characters is shuffled around

o   Rotor machines; old mechanical systems to perform substitution ciphers. Best-known example was the Enigma.

        1.4 SIMPLE XOR

o   non-secure symmetric encryption algorithm that has been in use in software products and US digital cell phone industry

        1.5 ONE-TIME PADS

o   perfect encryption scheme, rumored to have been in use in the hotline between the US and Soviet Union

o   impracticalities in key management; must be random, as long as the plaintext, cannot be reused, must remain secret

        1.6 COMPUTER ALGORITHMS

o   DES (Data Encryption Standard). Symmetric encryption algorithm.

RSA (based on Rivest, Shamir, Adleman). Public-key algorithm.

o   DSA (Digital Signature Algorithm). Public-key algorithm for signatures.

    • when evaluating cryptography, it is useful to understand the scope of numbers an algorithm is secure enough if breaking it takes longer than the universe remains

b) Give two examples of public key cryptography (other than PGP). Explain how public keys are used here.

       TLS

o   Transport Layer Security (TLS) is a protocol to secure communications, most visibly in web traffic (HTTPS).

o   In the beginning of the connection, a handshake with public key cryptography is used to establish cipher settings and session-spesific shared key.

(source: https://en.wikipedia.org/wiki/Transport_Layer_Security)

       Bitcoin

o   Bitcoin address = public key

o   The coin is a chain of digital signatures. Each owner transfers the coin to the next owner by digitally signing a hash of the previous transaction and the public key of the next owner.

(source: Nakamoto, Satoshi 2008: Bitcoin: A Peer-to-Peer Electronic Cash System. A colored HTML version.)
